## 适用场景

### Tesseract.js 适合：

✅ **打印文字**
- 教科书扫描件
- 试卷
- 讲义

✅ **小规模使用**
- 偶尔需要 OCR
- 图片数量少

✅ **完全免费**
- 不想配置 API
- 无配额限制

### PaddleOCR 适合：

✅ **所有文字类型**
- 打印文字
- 手写文字
- 复杂排版

✅ **大规模使用**
- 批量识别
- 图片数量多
- 需要高准确率

✅ **中英文混合**
- 双语教材
- 英文试卷（中文标注）

✅ **国际用户**
- 海外用户
- 不想用国内服务

### 百度 OCR 适合：

✅ **国内用户**
- 国内访问速度快
- 网络稳定

✅ **高精度识别**
- 对识别精度要求高
- 重要文档识别

✅ **中英文混合**
- 双语教材
- 英文试卷（中文标注）

✅ **企业级应用**
- 需要稳定服务
- 需要技术支持

## 配置对比

### Tesseract.js 配置

```env
# 无需配置，开箱即用
```

### PaddleOCR 配置

```env
NEXT_PUBLIC_PADDLE_OCR_API_URL=https://your-paddleocr-api.com/api/ocr
NEXT_PUBLIC_PADDLE_OCR_TOKEN=your_paddleocr_token_here
```

### 百度 OCR 配置

```env
NEXT_PUBLIC_BAIDU_OCR_ACCESS_TOKEN=your_baidu_access_token_here
```

## 使用建议

### 推荐方案

| 用户 | 推荐引擎 | 原因 |
|------|---------|------|
| **国内用户** | 百度 OCR | 国内访问速度快，识别效果优秀 |
| **国际用户** | PaddleOCR | 国际访问稳定，免费额度充足 |
| **学生** | 百度 OCR | 免费额度充足，识别效果好 |
| **教师** | 百度 OCR | 高精度识别，适合批改作业 |
| **开发者** | Tesseract.js | 无需配置，测试方便 |

### 切换方案

系统支持三个引擎，可以随时切换：

1. 打开智能导入页面
2. 开启「OCR 识别」
3. 选择 OCR 引擎
4. 开始识别

### 组合使用

建议组合使用：

- **首选百度 OCR**：国内用户默认使用
- **备用 PaddleOCR**：百度 OCR 配额用完时
- **备用 Tesseract.js**：完全免费，无配额限制

## 成本对比

### Tesseract.js

- **免费**：完全免费

- **无额外成本**：无需付费

- **免费额度**：100万次/月

- **成本预估**：

- **成本预估**：

## 总结

### Tesseract.js

✅ **优点**：
- 完全免费
- 无需配置
- 支持 100+ 语言
- 纯前端运行
- 无配额限制

❌ **缺点**：
- 识别效果一般
- 手写文字识别差
- 速度慢
- 首次使用需加载字库

### PaddleOCR

✅ **优点**：
- 识别效果好
- 支持手写文字
- 速度快
- 中英文混合识别好
- 100万次/月免费
- 国际访问稳定

❌ **缺点**：
- 需要 API Token
- 配额有限
- 国内访问可能慢
- 需要网络连接

### 百度 OCR

✅ **优点**：
- 国内访问速度快
- 识别效果好
- 支持手写文字
- 支持高精度识别
- 技术支持完善
- 企业级服务

❌ **缺点**：
- 需要 Access Token
- 配额较少（500次/天）
- 需要网络连接
- 额外配额需要付费

## 推荐配置

### 对于国内个人用户

推荐使用 **百度 OCR**：
- 国内访问速度快
- 识别效果好
- 免费额度充足（500次/天）
- 使用体验好

### 对于国内教育机构

推荐使用 **PaddleOCR + 百度 OCR 组合**：
- 主要使用百度 OCR（访问快）
- 备用 PaddleOCR（配额大）
- 降低成本

### 对于国际用户

推荐使用 **PaddleOCR**：
- 国际访问稳定
- 免费额度大（100万次/月）
- 识别效果好

### 对于开发者

推荐使用 **Tesseract.js**（测试） + **PaddleOCR/百度 OCR**（生产）：
- 测试时用 Tesseract.js（无需配置）
- 生产时用 PaddleOCR 或百度 OCR（效果好）

---

**总结**：
- **国内用户**：优先使用 **百度 OCR**
- **国际用户**：优先使用 **PaddleOCR**
- **测试开发**：使用 **Tesseract.js**
- **备用方案**：PaddleOCR 和 Tesseract.js
