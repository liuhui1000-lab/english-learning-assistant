## 适用场景

### Tesseract.js 适合：

✅ **打印文字**
- 教科书扫描件
- 试卷
- 讲义

✅ **小规模使用**
- 偶尔需要 OCR
- 图片数量少

✅ **完全免费**
- 不想配置 API
- 无配额限制

### PaddleOCR 适合：

✅ **所有文字类型**
- 打印文字
- 手写文字
- 复杂排版

✅ **大规模使用**
- 批量识别
- 图片数量多
- 需要高准确率

✅ **中英文混合**
- 双语教材
- 英文试卷（中文标注）

## 使用建议

### 推荐方案

| 场景 | 推荐引擎 | 原因 |
|------|---------|------|
| **日常使用** | PaddleOCR | 识别效果好、速度快 |
| **学生错题** | PaddleOCR | 支持手写文字 |
| **模拟卷** | PaddleOCR | 复杂排版识别好 |
| **单词表** | Tesseract.js | 打印文字即可 |
| **临时使用** | Tesseract.js | 无需配置 |

### 切换方案

系统支持两个引擎，可以随时切换：

1. 打开智能导入页面
2. 开启「OCR 识别」
3. 选择 OCR 引擎
4. 开始识别

### 组合使用

建议组合使用：

- **首选 PaddleOCR**：默认使用 PaddleOCR，识别效果好
- **备用 Tesseract.js**：PaddleOCR 配额用完时，切换到 Tesseract.js
- **测试对比**：同时使用两个引擎，对比识别结果

## 成本对比

## 总结

### Tesseract.js

✅ **优点**：
- 完全免费
- 无需配置
- 支持 100+ 语言
- 纯前端运行

❌ **缺点**：
- 识别效果一般
- 手写文字识别差
- 速度慢
- 首次使用需加载字库

### PaddleOCR

✅ **优点**：
- 识别效果好
- 支持手写文字
- 速度快
- 中英文混合识别好
- 100万次/月免费

❌ **缺点**：
- 需要 API Token
- 配额有限
- 需要网络连接

## 推荐配置

### 对于个人用户

推荐使用 **PaddleOCR**：
- 配额充足（100万次/月）
- 识别效果好
- 速度快
- 使用体验好

### 对于教育机构

推荐使用 **PaddleOCR + Tesseract.js 组合**：
- 主要使用 PaddleOCR
- 备用 Tesseract.js
- 降低成本

### 对于开发者

推荐使用 **两个引擎都测试**：
- 对比识别效果
- 对比速度
- 根据需求选择

---

**总结**：建议优先使用 **PaddleOCR**，识别效果好，免费额度充足。如配额用完，可切换到 **Tesseract.js**。
